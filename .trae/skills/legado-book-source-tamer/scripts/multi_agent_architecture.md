# 多智能体并行分析架构设计

## 架构概览

```
┌─────────────────────────────────────────────────────────────────┐
│                      主协调器 (Coordinator)                       │
│  1. 调用 batch_fetcher.py 并行抓取所有页面                          │
│  2. 启动多个智能体并行分析                                          │
│  3. 汇总结果，生成书源 JSON                                        │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                   batch_fetcher.py (并行抓取)                     │
│  同时抓取: 首页、搜索页、发现页、详情页、目录页、正文页                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌───────────┬───────────┬───────────┬───────────┬───────────┐
│ Agent 1   │ Agent 2   │ Agent 3   │ Agent 4   │ Agent 5   │
│ 搜索页    │ 发现页    │ 详情页    │ 目录页    │ 正文页    │
│ 分析器    │ 分析器    │ 分析器    │ 分析器    │ 分析器    │
└───────────┴───────────┴───────────┴───────────┴───────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      结果汇总 & 书源生成                           │
└─────────────────────────────────────────────────────────────────┘
```

## 智能体分工

| 智能体 | 输入 | 分析任务 | 输出 |
|--------|------|----------|------|
| 搜索页分析器 | 首页HTML、搜索页HTML | 发现搜索API、参数、编码 | searchUrl, ruleSearch |
| 发现页分析器 | 分类页HTML | 分类URL格式、书籍列表结构 | exploreUrl, ruleExplore |
| 详情页分析器 | 详情页HTML | 书名、作者、封面、简介 | ruleBookInfo |
| 目录页分析器 | 目录页HTML | 章节列表结构 | ruleToc |
| 正文页分析器 | 正文页HTML | 正文内容、下一页 | ruleContent |

## 实现方式

使用 Task 工具启动多个 search 智能体并行工作：

```python
# 伪代码示意
results = batch_fetcher.fetch_all(urls)  # 并行抓取

# 启动多个智能体并行分析
agents = [
    Task(agent="search", query="分析搜索页...", input=results['search']),
    Task(agent="search", query="分析发现页...", input=results['category']),
    Task(agent="search", query="分析详情页...", input=results['detail']),
    Task(agent="search", query="分析目录页...", input=results['toc']),
    Task(agent="search", query="分析正文页...", input=results['content']),
]

# 并行执行所有智能体
# 汇总结果生成书源
```

## 优势

1. **真正的并行** - 抓取和分析都并行
2. **专注分析** - 每个智能体只负责一种页面类型
3. **可扩展** - 可以轻松添加更多分析任务
4. **容错性** - 单个智能体失败不影响其他
