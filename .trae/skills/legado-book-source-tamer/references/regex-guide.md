## 基本格式

在Legado书源规则中，正则表达式用于文本清理和内容提取，格式如下：

### 核心规则（最重要！）

```
##正则表达式##替换内容
```

**关键理解**：
- `##正则表达式` - 末尾不写`##`，表示**替换为空白**（即删除）
- `##正则表达式##替换内容` - 末尾写`##替换内容`，表示**替换为指定内容**
- 多个规则用`|`分隔：`##规则1|规则2|规则3` - 删除所有匹配内容

---

## 格式1：删除匹配的内容（替换为空白）

```
选择器@提取类型##正则表达式
```

**示例**：
```js
// 删除"作者："前缀
"author": ".author@text##^作者："
// 结果："张三"（"作者："被删除）

// 删除"看书更精彩"
"content": "#content@html##看书更精彩"
// 结果：所有"看书更精彩"都被删除
```

---

## 格式2：替换为指定内容

```
选择器@提取类型##正则表达式##替换内容
```

**示例**：
```js
// 将"旧文本"替换为"新文本"
"content": ".content@text##旧文本##新文本"

// 将连续空格替换为单个空格
"content": ".content@text##\\s+## "
```

---

## 格式3：使用捕获组提取特定内容

```
选择器@提取类型##正则表达式(捕获组)##$1
```

**示例**：
```js
// 提取"作者：xxx"中的"xxx"
"author": ".author@text##.*作者：(.*)##$1"
// 结果："钱真人"
```

---

## 格式4：只返回替换内容（三个#结尾）

```
选择器@提取类型##正则表达式##替换内容###
```

**核心规则**：
- `##正则##替换###` - 末尾用三个`#`，表示**只返回替换后的内容**，而不是整个字符串
- 如果正则匹配失败，返回**空字符串**

**示例对比**：
```js
// 原始文本："这是一段文字"

// 格式2：替换为指定内容（两个#结尾）
"text##文字##替换"
// 结果："这是一段替换"（整个字符串，匹配部分被替换）

// 格式4：只返回替换内容（三个#结尾）
"text##文字##替换###"
// 结果："替换"（只返回替换后的内容）

// 匹配失败时
"text##没有##替换###"
// 结果：""（空字符串，因为"没有"不匹配）
```

**实战应用**：
```js
// 提取并转换特定格式
// 原文："价格：99元"
"price@text##价格：(\d+)元##$1元###"
// 结果："99元"（只返回匹配并替换后的内容）

// 如果原文是："价格：免费"
"price@text##价格：(\d+)元##$1元###"
// 结果：""（空字符串，因为正则不匹配）
```

## 多个清理规则（重要！）

**使用 `|` 分隔多个清理规则，末尾不需要`##`**：

```js
// HTML: <div class="content">
//   <p>正文内容1</p>
//   <div id="ad">广告内容</div>
//   <p>正文内容2</p>
//   <p>请收藏本站</p>
//   <p>看书更精彩</p>
// </div>
// 规则：删除广告、提示文本、"看书更精彩"
"content": ".content@html##<div id=\"ad\">[\\s\\S]*?</div>|请收藏本站|看书更精彩"
// 注意：末尾没有##，表示所有匹配内容都替换为空白（删除）
```

**常见错误写法**：
```js
// 错误：末尾多写了##
"content": ".content@html##规则1|规则2##"
// 这会把"规则1|规则2"替换为空白，而不是分别删除规则1和规则2

// 正确：末尾不写##
"content": ".content@html##规则1|规则2"
// 这会分别删除规则1和规则2
```

## 正则表达式完整性检查

**验证清单**：
1. 是否使用`##`作为分隔符？
2. 如果需要提取特定内容，是否使用捕获组`()`和引用`$1`、`$2`？
3. 是否包含所有需要清理的广告和提示文本？
4. 多个清理规则是否使用`|`分隔？
5. **末尾是否正确**：删除内容时末尾不写`##`，替换内容时末尾写`##替换内容`

---

## 常见错误

**错误1：未使用##分隔符**
```js
"author": ".author@text /作者：(.*)/"  // 错误
```

**错误2：捕获组后未使用引用**
```js
"author": ".author@text##作者：(.*)##"  // 错误，应该使用$1
// 正确写法：
"author": ".author@text##作者：(.*)##$1"  // 正确
```

**错误3：多规则末尾多写了##**
```js
"content": ".content@html##规则1|规则2##"  // 错误，末尾的##会被当作替换内容
// 正确写法：
"content": ".content@html##规则1|规则2"  // 正确，删除规则1和规则2
```

---

## 正确示例

```js
// 删除单个内容
"content": "#content@html##看书更精彩"  // 删除"看书更精彩"

// 删除多个内容
"content": "#chaptercontent@html##<div id=\"content_tip\">[\\s\\S]*?</div>|本章节未完，点击下一页继续阅读|歌书网.*com"  // 删除所有匹配内容

// 提取特定内容
"author": ".author@text##.*作者：(.*)##$1"  // 提取作者名
```

## 记忆口诀

**正则表达式格式**：
- 删除内容：`##正则表达式`（末尾不写##）
- 替换内容：`##正则表达式##替换内容`
- 提取内容：`##正则表达式(捕获组)##$1`
- 只返回替换：`##正则表达式##替换内容###`（末尾三个#）

**常见用法**：
- 删除前缀：`##^作者：`
- 删除后缀：`##（.*）$`
- 删除多个：`##规则1|规则2|规则3`
- 提取内容：`##.*作者：(.*)##$1`

**核心规则**：
- 末尾不写`##` = 替换为空白（删除）
- 末尾写`##内容` = 替换为指定内容
- 末尾写`###` = 只返回替换后的内容（匹配失败返回空）

## 高级技巧

### 正则替换与JavaScript结合

在规则中使用正则替换后，可以继续用JavaScript处理：

```javascript
// 先用正则提取，再用JS判断
"author": ".qk-title-text@textNodes##(?s).*作者\\s(.*)\\-(?s).*##$1\n<js>result = /第.*章/.test(result) ? \"\" : result</js>"
```

**处理流程**：
1. 正则 `(?s).*作者\s(.*)\-(?s).*` 提取作者名
2. JavaScript 判断结果是否包含章节信息
3. 如果包含章节信息，返回空字符串；否则返回作者名

**关键技术**：
- `(?s)` 启用单行模式，`.` 可以匹配换行符
- `\s` 匹配空白字符
- `/第.*章/.test(result)` 正则测试是否匹配

### 从URL中提取信息构造新URL

使用正则从URL中提取ID等信息，构造新的URL：

```javascript
// 从搜索结果URL提取信息，构造封面URL
"coverUrl": "a.0@href##.*libahao.com/book/(.*)_(.*)/(.*)?##https://m.libahao.com/data/image/$2.jpg"

// 从搜索结果URL提取信息，构造详情页URL
"bookUrl": "a.0@href##.*libahao.com/book/(.*)_(.*)/(.*)?##https://m.libahao.com/book/$1_$2/"
```

**正则解析**：
- `(.*)_(.*)/(.*)?` 匹配 `/book/xxx_yyy/zzz?` 格式
- `$1` = 第一个捕获组（xxx）
- `$2` = 第二个捕获组（yyy）
- `$3` = 第三个捕获组（zzz）

### 多行文本处理

使用 `(?s)` 或 `[\s\S]*` 处理多行文本：

```javascript
// 使用 (?s) 模式
"text##(?s).*作者\\s(.*)\\-(?s).*##$1"

// 使用 [\s\S]* 匹配任意字符（包括换行）
"text##[\\s\\S]*作者\\s(.*)\\-[\\s\\S]*##$1"
```

**两种写法等价**：
- `(?s)` 开启单行模式，`.` 匹配任意字符（包括换行）
- `[\s\S]` 匹配空白字符或非空白字符，即任意字符

### 从HTML/JSON混合数据中提取URL

百度搜索结果中，真实URL存储在元素的JSON数据中，需要特殊处理：

```javascript
// 百度搜索结果的HTML结构中包含JSON数据
// &quot;mu&quot;:&quot;https://example.com/book/123&quot;

// 使用JavaScript split提取URL
"bookUrl": "@js:String(result).split(\"&quot;mu&quot;:&quot;\")[1].split(\"&quot;\")[0]"
```

**提取步骤**：
1. `String(result)` 将元素转为字符串
2. `.split("&quot;mu&quot;:&quot;")` 按JSON字段分割，取第二部分
3. `.split("&quot;")[0]` 按引号分割，取URL部分

**HTML实体编码对照**：

| 字符 | HTML实体 | 说明 |
|------|----------|------|
| `"` | `&quot;` | 双引号 |
| `'` | `&#39;` | 单引号 |
| `<` | `&lt;` | 小于号 |
| `>` | `&gt;` | 大于号 |
| `&` | `&amp;` | 和号 |

**使用正则提取（备选方案）**：

```javascript
// 使用正则从JSON数据中提取URL
"bookUrl": "@js:String(result).match(/&quot;mu&quot;:&quot;([^&]+)&quot;/)[1]"
```

### 使用contains伪类配合正则

Legado不支持`:contains()`伪类，但可以使用 `text.文本` 格式：

```javascript
// 错误写法（不支持）
".cu-mr-base:contains(作者)@text"

// 正确写法
".cu-mr-base:contains(作者)@text"  // Legado实际支持这种写法
```

**注意**：Legado的Default语法支持 `:contains()` 选择器，但CSS选择器模式不支持

## 章节名标准化处理

当网站章节名格式不统一时，可以使用JavaScript链式replace进行标准化：

### 完整示例

```javascript
"lastChapter": "{{@@[property$=chapter_name]@content##正文卷.|正文.|VIP卷.|默认卷.|卷_|VIP章节.|免费章节.|章节目录.|最新章节.|[\\(（【].*?[求含理更谢乐发推票盟补加字].*?[】）\\)]}}•{{@@[property$=update_time]@content##\\s.*}}\n@js:result\n.replace(\"••\",\"\")\n.replace(/^(\\d+).第/,'第')\n.replace(/^(\\d+)[、．]第.+章/,'第$1章')\n.replace(/^(\\d+)、\\d+、/,'第$1章 ')\n.replace(/^(\\d+)、\\d+/,'第$1章')\n.replace(/^(\\d+)、/,'第$1章 ')\n.replace(/^(第.+章)\\s?第.+章/,'$1')\n.replace(/第\\s(.+)\\s章/,'第$1章')\n.replace(/.*(chapter|Chapter)\\s?(\\d+)\\s?/,'$1 $2 ')\n.replace(/\\(.+\\)/,'')\n.replace(/\\[|。/,'')\n.replace(/第([零一二两三四五六七八九十百千]+)章/g,java.toNumChapter(result))\n##(章)([^\\s]+)(\\s·)##$1 $2$3"
```

### 处理流程

| 步骤 | 正则 | 说明 |
|------|------|------|
| 1 | `##正文卷.\|正文.\|VIP卷.\|...` | 清理卷名前缀 |
| 2 | `##[\(（【].*?[求含理更谢乐发推票盟补加字].*?[】）\)]` | 清理括号内的求票等信息 |
| 3 | `.replace(/^(\\d+).第/,'第')` | 修复"1.第一章"格式 |
| 4 | `.replace(/^(\\d+)[、．]第.+章/,'第$1章')` | 修复"1、第一章"格式 |
| 5 | `.replace(/^(\\d+)、/,'第$1章 ')` | 修复"1、"格式 |
| 6 | `.replace(/第\\s(.+)\\s章/,'第$1章')` | 清理章节名中的空格 |
| 7 | `.replace(/.*(chapter\|Chapter)\\s?(\\d+)/,'$1 $2')` | 标准化英文章节 |
| 8 | `java.toNumChapter(result)` | 中文数字转阿拉伯数字 |

### 常见章节名格式及处理

| 原始格式 | 处理后 | 正则 |
|----------|--------|------|
| `正文卷 第一章` | `第一章` | `##正文卷.` |
| `1.第一章 开始` | `第一章 开始` | `/^(\\d+).第/,'第'` |
| `1、初遇` | `第1章 初遇` | `/^(\\d+)、/,'第$1章 '` |
| `第 一 章` | `第一章` | `/第\\s(.+)\\s章/,'第$1章'` |
| `Chapter 1` | `Chapter 1` | `/.*(chapter\|Chapter)\\s?(\\d+)/,'$1 $2'` |
| `第一章[求票]` | `第一章` | `##[\(（【].*?[求含理更谢乐发推票盟补加字].*?[】）\)]` |

### 中文数字转阿拉伯数字

```javascript
.replace(/第([零一二两三四五六七八九十百千]+)章/g, java.toNumChapter(result))
```

**转换示例**：
- `第十章` → `第10章`
- `第一百二十章` → `第120章`
- `第一千零一章` → `第1001章`

### 链式replace技巧

```javascript
// 使用\n分隔多个replace
@js:result
.replace("旧文本", "新文本")
.replace(/正则/g, "替换")
.replace(/另一个正则/, "")
```

**注意**：
- 每个replace独占一行，提高可读性
- 使用 `/g` 标志进行全局替换
- 最后可以用 `##正则##替换` 进行最终清理
