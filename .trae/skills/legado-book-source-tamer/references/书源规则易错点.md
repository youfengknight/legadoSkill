# 书源规则易错点

> 更新时间: 2026-03-21
> 描述: 记录书源编写过程中常见的错误和注意事项

## 正文规则：不同源类型的区别

做书源、漫画源、听书源(音乐源)、视频源，在正文规则是有区别的：

### 书源（文本）

- **获取内容**：txt 或 html 文本
- **规则写法**：`@txt` 或 `@html`**示例**：
  ```json
  "content": "@txt"
  // 或其他格式的文本，具体看源规则
  "content": "@html"
  ```

### 漫画源（图片）

- **获取内容**：图片链接
- **规则写法**：用 JS 拼成 `<img>` 标签
- **注意**：如果图片链接需要 header 属性，也要加上
- **示例**：
  ```javascript
  // 简单情况
  "<img src=\"" + result + "\">"

  // 需要header的情况，这里还是有问题，需要看实例
  "<img src=\"" + result + ",{header={'Referer':'https://example.com'}}\">"

  // 多图拼接
  var imgs = JSON.parse(result).data;
  var html = '';
  for (var i = 0; i < imgs.length; i++) {
      html += '<img src="' + imgs[i] + '">';
  }
  html;
  ```

### 听书源/音乐源（音频）

- **获取内容**：音频文件链接
- **规则写法**：直接返回字符串链接即可
- **示例**：
  ```javascript
  // 直接返回音频URL
  var audioUrl = result.match(/url\d+\s*=\s*'([^']+)'/);
  if (audioUrl) return audioUrl[1] + '.mp3';
  return '没有找到链接';
  ```

### 视频源

- **获取内容**：视频链接
- **规则写法**：使用阅读自带的打开浏览器函数
- **示例**：
  ```javascript
  // 使用 startBrowser 打开视频
  java.startBrowser(videoUrl, title);
  // 或返回视频链接让阅读处理
  videoUrl;
  ```

### 快速对照表

| 源类型 | bookSourceType | 正文返回格式      | 示例                            |
| --- | -------------- | ----------- | ----------------------------- |
| 书源  | 0              | 文本/HTML     | `@txt` 或 `@html`等等，具体看源规则              |
| 听书源 | 1              | 音频URL字符串    | `https://xxx.mp3`             |
| 漫画源 | 2              | `<img>` 标签  | `<img src="...">`             |
| 视频源 | 3              | 视频URL或浏览器调用 | `java.startBrowser(...)` |

## 内置变量

### result

- **用途**: 表示当前页面的 HTML 内容
- **错误**: 不要使用 `baseStr`，这个变量不存在
- **正确**: 使用 `result` 获取当前页面 HTML

### baseUrl

- **用途**: 当前请求的 URL
- **类型**: String

## 变量声明

### 禁止使用 var

- **原因**: Legado 内部已有同名变量，使用 `var` 会覆盖内置变量
- **错误示例**:
  ```javascript
  var result = baseStr.match(...);  // 错误！
  var m = url.match(...);         // 错误！
  ```
- **正确示例**:
  ```javascript
  var m = result.match(...);  // 正确，使用自定义变量名
  ```

### 变量命名建议

- 避免与内置变量重名：`result`、`baseUrl`、`java`、`book`、`chapter`、`source`、`cache`、`cookie`、`title`、`src`、`nextChapterUrl`、`isFromBookInfo`
- 推荐使用简短变量名：`m`、`html`、`url`、`mp3`、`ext`

## 音频源处理

### 直接从 setMedia 获取完整 URL

- **错误**: 尝试拼接 `url` + `murl`
  ```javascript
  var url = playerHtml.match(/url\d+\s*=\s*'([^']+)'/);  // url 变量可能是空的！
  var ext = playerHtml.match(/murl\d+\s*=\s*'([^']+)'/);
  return url[1] + ext[1];  // 错误！
  ```
- **正确**: 直接匹配 `setMedia({ mp3: '...' })` 中的完整 URL
  ```javascript
  var mp3 = playerHtml.match(/mp3:\s*'([^']+)'/);
  if (mp3) return mp3[1];
  return '';
  ```

## 目录跳转处理

### 多级跳转场景

- **场景**: 详情页 → `/dir/` 中间页 → `/bookdir/` 真实目录页
- **处理方法**: 使用 JS 请求中间页，提取最终链接
  ```javascript
  var m = result.match(/href="([^"]*dirurl[^"]*)"/i);
  if (!m) return '';
  var dirUrl = 'https://www.example.com' + m[1];
  var html = java.ajax(dirUrl);
  var bookdir = html.match(/href="([^"]*bookdir[^"]*)"/i);
  if (bookdir) return bookdir[1];
  return '';
  ```

## CSS 选择器

### id 和 class 选择器兼容

- **错误**: 只使用 `#id` 选择器
  ```json
  "chapterList": "@css:#playlist li"  // 只能匹配 id="playlist"
  ```
- **正确**: 使用 `.class` 选择器兼容两种情况
  ```json
  "chapterList": "@css:.playlist li"  // 同时匹配 id="playlist" 和 class="playlist"
  ```

## 正则表达式

### 正则转义错误

- **场景**: Legado 书源 JSON 里的正则，反斜杠不需要额外转义
- **错误**: 在 JSON 字符串中双重转义反斜杠
  ```json
  "content": "@js:\nresult.replace(/\\\\\\[|\\\\\\]/g, '')"
  ```
- **正确**: 直接写正则，不需要额外转义
  ```json
  "content": "@js:\nresult.replace(/\\[|\\]/g, '')"
  ```
- **来源**: [skill-tips-guide.md](skill-tips-guide.md#5-正则替换速记)

### 匹配包含特定字符的属性

- **场景**: 匹配 `dirurl`、`bookdir` 等包含特定关键词的属性值
- **错误**: 过于严格的匹配
  ```javascript
  /href="\/dir\/yangguiweihuo[^"]*"/i  // 过于严格
  ```
- **正确**: 使用模糊匹配
  ```javascript
  /href="([^"]*dirurl[^"]*)"/i  // 模糊匹配，更灵活
  ```
